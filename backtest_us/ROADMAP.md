# NASDAQ 퀀트 — 로드맵 & 방향

> 벤치마크 철학: 김민겸(2025 WorldQuant IQC 단독우승)
> 소수정예 · 안정적 우상향(Sharpe/MDD) · 레짐 분산 · AI=조교

## 확정된 방향 (2026-06)
- **자동화 = L1 (반자동)**: 시스템이 주 1회 바스켓 제안 → **사람 승인** → KIS 집행.
  AI는 복기·리포트만, 매매 결정은 사람.
- **신·구 관계 = 수렴**: 검증 끝나면 `kis/` 라이브 봇의 즉흥 진입규칙을
  `backtest_us`에서 **검증된 시그널로 교체**. 봇은 "집행기"로만 남김.
- **시장 = 미장(NASDAQ) 집중**: 첫 알파는 미장 한 곳. (한국장은 이후)

## 합격 기준 (돌리기 전에 못박는다 — 결과에 끌려다니지 않기)
첫 알파(Clenow 미장)는 아래를 **모두** 만족해야 채택:
1. **QQQ 매수후보유 대비 Sharpe 더 높음**
2. **QQQ 대비 MDD 더 얕음** (또는 비슷한데 Sharpe 우위)
3. 파라미터(lookback/top_n)를 흔들어도 성과 유지 (특정값 과적합 아님)

하나라도 못 넘으면 → 알파 수정 or 폐기. **PnL 크기는 판단 기준 아님.**

## 작업 큐
```
[1] 검증      backtest_us 로 Clenow 미장 백테스트            ← 현재 단계
              실행: GitHub Actions(폰) 또는 맥 로컬
[2] 견고성    walk-forward + lookback/top_n 민감도 점검
[3] 시그널화  검증된 알파 → 주1회 "바스켓+비중" JSON
              여기서 kis/ 즉흥 진입규칙 교체
[4] L1 집행   텔레그램 제안 → 사람 승인 → KIS 주문
[5] 조교 루프 intelligence/ 가 체결·저널·복기 리포트
[6] 2번째 알파 리버전 sleeve 추가 (레짐 분산) — [1] 합격 후에만
```

## 폰 ↔ 맥 ↔ Actions 분업
| 환경 | 네트워크 | 역할 |
|------|---------|------|
| **폰 (Claude on web)** | GitHub/PyPI만 | 코드 작성·엔진 검증(합성)·커밋·**Actions 트리거** |
| **GitHub Actions 러너** | 열림 | stooq 데이터 받아 **실제 백테스트** → 결과 커밋 + 텔레그램 |
| **맥 (로컬, 선택)** | 열림 | 대량 실험·탐색 시 |

→ `.github/workflows/nasdaq_backtest.yml` 로 **폰만으로 전체 루프 가능**.
   맥은 있으면 편하지만 필수 아님.

## 현재 상태

- ⬜ [1] 실제 데이터 백테스트 결과 (Actions/맥에서 실행 대기)
